# 第4章 · 模型评估

训练出一个模型并不意味着工作结束。事实上，如果不对模型进行正确的评估，你训练出来的可能是一个看起来很完美却毫无实际价值的模型。本章讲解如何评估机器学习模型的性能，以及如何诊断和解决过拟合问题。

## 4.1 为什么需要评估？

在使用最速下降法和随机梯度下降法训练模型时，我们不断地更新参数，让目标函数的值越来越小（或越来越大）。这个过程结束后，参数确实是为训练数据"量身定制"的。

但这引出一个关键问题：**训练数据拟合得好，不代表对新数据也能预测得好。**

回想第2章的多项式回归：给定 10 个数据点，9 次多项式可以完美地穿过每一个训练点（训练误差接近 0），但它在新数据上的表现可能极其糟糕——曲线在数据点之间剧烈摆动，毫无泛化能力。

这就是**过拟合**——模型记住了训练数据的噪声，而不是学到了真正的规律。

## 4.2 交叉验证

### 基本思想

将全部数据分为两部分：

- **训练集（Training Set）**：用来更新参数
- **测试集（Test Set）**：用来评估模型，**绝不能用于训练**

常见的分割比例是 7:3 或 8:2（训练:测试）。

### 回归模型的评估指标：MSE

对于回归问题，最常用的评估指标是**均方误差（Mean Square Error, MSE）**：

$$\text{MSE} = \frac{1}{n} \sum_i \big[ y^{(i)} - f_\theta(x^{(i)}) \big]^2$$

其中 $n$ 是**测试数据**的数量，$i$ 是测试数据（不是训练数据）的索引。

注意这个公式和训练时使用的目标函数完全一样，只是计算的数据不同——MSE 使用的是**测试数据**，是模型从未见过的数据。

MSE 越小，模型对新数据的预测能力越强。通过比较模型在训练集和测试集上的 MSE，可以判断是否存在过拟合：
- 训练 MSE ≈ 测试 MSE（都很小）→ 模型良好
- 训练 MSE ≪ 测试 MSE → 过拟合
- 训练 MSE 和测试 MSE 都很大 → 欠拟合

### 分类模型的评估指标：Accuracy

对于分类问题，最简单直观的指标是**准确率（Accuracy）**：

$$\text{Accuracy} = \frac{\text{预测正确的样本数}}{\text{总样本数}}$$

为了更好地分析分类结果，引入**混淆矩阵（Confusion Matrix）**：

| | 真实：正类 | 真实：负类 |
|---|---|---|
| **预测：正类** | True Positive (TP) | False Positive (FP) |
| **预测：负类** | False Negative (FN) | True Negative (TN) |

四个关键数字：
- **TP**（真正例）：真实为正、预测也为正——分类正确 ✓
- **TN**（真负例）：真实为负、预测也为负——分类正确 ✓
- **FP**（假正例）：真实为负、预测为正——分类错误 ✗（误报）
- **FN**（假负例）：真实为正、预测为负——分类错误 ✗（漏报）

### 精确率与召回率

Accuracy 在**数据不均衡**时会失效。假设 100 个样本中 95 个是负类，模型偷懒把全部样本都预测为负类，Accuracy 也有 95%——但它完全没有识别任何正类。

这时需要两个补充指标：

**精确率（Precision）**：

$$\text{Precision} = \frac{\text{TP}}{\text{TP} + \text{FP}}$$

含义：在所有被预测为正类的样本中，真正是正类的比例。衡量"少犯误报的错误"。

**召回率（Recall）**：

$$\text{Recall} = \frac{\text{TP}}{\text{TP} + \text{FN}}$$

含义：在所有真实为正类的样本中，被正确识别出来的比例。衡量"少漏掉正类"。

**精确率和召回率的权衡**：这两个指标通常是矛盾的——提高阈值（更谨慎地判定为正类）会提高精确率但降低召回率；降低阈值（更激进地判定为正类）会提高召回率但降低精确率。

**F 值（F-measure）** 综合两者：

$$F = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}$$

F 值是精确率和召回率的**调和平均**，当两者都很高时 F 值才会高。如果一个很高一个很低，F 值会被拉低。

## 4.3 过拟合与欠拟合

### 过拟合（Overfitting）

**症状**：训练误差很小，测试误差很大。

**原因**：
- 模型过于复杂（参数太多）
- 训练数据太少
- 训练时间太长
- 数据有噪声，模型把噪声也记住了

**解决策略**：
- 增加训练数据量
- 简化模型（减少参数/降低特征维度）
- 使用正则化
- 早停（Early Stopping）：在测试误差开始上升时停止训练

### 欠拟合（Underfitting）

**症状**：训练误差和测试误差都很大。

**原因**：
- 模型太简单（无法捕捉数据的真实模式）
- 特征工程不足（没有提供足够的有效特征）
- 训练不充分

**解决策略**：
- 增加模型复杂度（如增加多项式次数）
- 添加更多有效特征
- 减少正则化强度
- 增加训练时间

### 学习曲线

**学习曲线**是将训练过程中训练误差和测试误差随训练次数（epoch）变化的情况画成图：

- 训练误差曲线：在训练数据上计算的误差
- 测试误差曲线：在测试数据上计算的误差

**典型形态解读**：
- 两条曲线都下降且最终接近 → 模型学习良好，没有过拟合
- 训练误差持续下降但测试误差上升（U 型）→ 过拟合开始
- 两条曲线都高且接近 → 欠拟合

## 4.4 正则化：防止过拟合的系统方法

### 什么是正则化

正则化（Regularization）在目标函数中加入额外项，**惩罚过大的参数值**，迫使模型选择更简单的函数。

修改后的目标函数：

$$E_{\text{reg}}(\theta) = E(\theta) + \frac{\lambda}{2} \sum_j \theta_j^2$$

其中：
- $E(\theta)$ 是原始目标函数（如 MSE 或对数似然的负数）
- 第二项是**正则化项**（也称 L2 正则化 / 权重衰减）
- $\lambda$（lambda）是**正则化参数**，控制惩罚强度

### 正则化为何有效

直观解释：过拟合的模型通常参数值很大（曲线上下剧烈摆动），正则化通过惩罚大的参数值，倾向于选择参数值更小的平滑函数。这相当于在告诉模型："在拟合数据的同时，尽量保持函数简单"。

就像**奥卡姆剃刀原则**：在能解释数据的多个模型中，最简单的往往是最好的。

### 选择正则化强度 $\lambda$

$\lambda$ 的取值非常关键：
- $\lambda = 0$ → 没有正则化，等价于原始训练
- $\lambda$ 太小 → 正则化效果不明显
- $\lambda$ 适中 → 有效防止过拟合
- $\lambda$ 太大 → 模型过于简单，可能欠拟合（参数趋近于零）

实践中通常通过交叉验证来选择最优的 $\lambda$ 值。

### 正则化与参数更新

在梯度下降中加入正则化后，参数更新公式变为：

$$\theta_j := \theta_j - \eta \cdot \left[ \frac{\partial E}{\partial \theta_j} + \lambda \cdot \theta_j \right]$$

$$= (1 - \eta \cdot \lambda) \cdot \theta_j - \eta \cdot \frac{\partial E}{\partial \theta_j}$$

可以看到，正则化相当于每一步都先把参数**缩小一点**（乘以 $(1 - \eta \cdot \lambda)$），然后再按梯度更新。这也解释了"权重衰减"名称的由来。

## 4.5 评估流程总结

一个完整的模型评估流程应该是：

1. **划分数据**：将数据集分成训练集和测试集（有条件的话，再分出验证集用于调参）
2. **训练模型**：只在训练集上更新参数
3. **评估指标**：在测试集上计算 MSE（回归）或 Accuracy/Precision/Recall（分类）
4. **诊断问题**：
   - 过拟合？→ 加正则化 / 简化模型 / 加数据
   - 欠拟合？→ 加特征 / 加复杂度 / 减正则化
5. **绘制学习曲线**：观察训练和测试误差的变化趋势
6. **迭代改进**：根据诊断结果调整模型，重复评估
